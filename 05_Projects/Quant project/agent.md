## Project Rules

1. 不得把未驗證模型結果寫成強結論。
2. 所有策略、模型、投資觀點都必須保留假設與限制。
3. 若沒有回測，不得宣稱策略有效。
4. 若是市場、金融、公司資訊，需注意時效性。
5. 若內容可長期複用，請整理成 Atomic Note 草稿。
6. 若內容只對本 project 有效，留在 project output。
7. 不得直接修改正式 MOC，除非使用者要求。
